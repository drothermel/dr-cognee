01

The brief & the framework

written before any research, so findings had a structure to land in — 00-framework.md

The user fixed the scope up front: viz for live drc runs plus result exploration (graph, knowledge, report), for one power user, as a local web app, on a very wide/tall monitor. The load-bearing framing: during a run the agent operates drc — the human's job is oversight, trust, and steering. The framework then pinned three vocabularies everything downstream reuses: data shapes D1–D10, lifecycle moments M1–M7, and a six-criterion selection rubric.

The two moments

Live observability

Watching a run as it happens: harvest fan-out, status transitions, distill output landing, synthesis evolving, ingest/cognify progress, the graph growing. The human asks: is coverage good? is it advancing? what did it just learn? where do I redirect it?

Result exploration

After (or between) runs: the Cognee graph's structure, the knowledge it carries (entities, claims, provenance), and the report produced from the research question.

Two structural observations that shaped every design

The workspace is watchable — everything but the hosted graph is plain files; a watcher + SSE gets live updates nearly for free. And D5 is underexploited — entities in distilled records form a co-occurrence graph before Cognee ever runs, with perfect provenance.

The three eventual picks cover M1–M4+M7-adjacent (mission), M5 (atlas), M6 (dossier) — the distinctness rule at work.

The selection rubric score 1–5, pick a portfolio

  1. Decision value — does it change what the human does next during/after a run?
  2. Data availability — consumes data we provably have (D7 was gated on R8's findings).
  3. Build cost — weekend-buildable by one person on the chosen stack.
  4. Maintenance drag — survives pipeline evolution without constant care.
  5. Wide-canvas payoff — genuinely benefits from the big monitor.
  6. Distinctness — portfolio-level: the 3 picks must cover different moments; at least one live, one exploration.

Plus a design template every chosen use case had to follow: goal → problems & constraints → UX flow → UI wireframe → operationalization (v0/v1 phases, risks, estimates).

03

Findings gallery

the discoveries that actually changed the designs

Six research findings set the design space; four critique catches — all made by reading the real code and real workspaces, not the design prose — reshaped it before a line of build code exists.

From the research wave · research/ + 03-synthesis.md
R8 · live-verified

The graph is fully reachable — one call

GET /api/v1/datasets/{id}/graph verified against the live hosted tenant: 3,033 nodes / 13,189 edges as JSON, with node types, descriptions, belongs_to_set, importance weights, labeled edges. Plus /schema/inventory (free overview layer), /visualize (the d3 baseline to beat), and verbose:true search returning triplet objects. INSIGHTS search no longer exists — nothing gets designed against it.

consequence The atlas went from "gated on an open question" to "designed against verified payload shapes." D5 demoted from primary fallback to pre-cognify preview.
R1 + R7 · convergence

One architecture serves everything

Two independent agents converged: a single local FastAPI process, zero storage, watchfiles → SSE → htmx fragment updates, graph libs as CDN ES modules, Cognee proxied server-side. The load-bearing plumbing rule: never tail a rewritten file — reparse sources.jsonl on change, diff keyed snapshots (id → hash), emit semantic transition events.

consequence The watcher mints events.jsonl — solving the no-timestamps gap with zero pipeline changes, and powering feed, badges, timeline, saturation curve, and future replay.
R3 · empirical

Hairballs fail past ~200 nodes

Global graph views empirically collapse beyond ~200 nodes (the Obsidian evidence); the field replaced overview-first with "search, show context, expand on demand." The ego view is the workhorse: DOI-bounded 30–50 node neighborhoods, ranked expansion, structured layout — not free physics. Provenance-on-click is the highest-value feature for trusting LLM-built graphs.

consequence Atlas lands on search + community meta-node overview, never the full graph — exactly what Cognee's own everything-at-once visualize doesn't do.
R4 · controlled studies

Diffs beat animation

For change comprehension, static two-snapshot diffs with synchronized positions outperform growth animation in controlled studies. Animation earns its keep only for live "what just appeared" monitoring and replay theater — not analysis.

consequence Cognify-to-cognify comparison shipped as a union diff + "most rewired entities" list; standalone replay (C6) was cut with evidence, not vibes.
R5 · the divider

Passage anchors make or break verification

Passage-anchored citations (source_id + span) are the single biggest divider between fast and painful claim-checking. The detail ladder: chip → hover card (target: 80% of verifications end here) → distilled record → full content. Valence badges (USE / VENDOR / SECONDHAND) operationalize drc's own quality rule; un-cited claims get explicit gap markers.

consequence Today's records lack char spans — the dossier ships fuzzy-matching as the fallback tier and upgrades the distill quote-span change to a prerequisite for exact highlights.
R6 · reframe

Directives, not statuses

During a run the human is a steerer, not an operator: triage is keyboard verbs + auto-advance + zero-target queues, and the human's output is structured directives the agent consumes between iterations — not status flips. Nothing on the market does human→agent steering directives well.

consequence directives.jsonl became the mission's one genuinely novel mechanism — later pulled forward to v1 when the UX critic showed the verb loop didn't persist without it.
What the critics caught — code vs design · critique/ · checked against 07-07-tool-graph + 4 real workspaces
catch · threatening

Non-atomic writes → torn reads

SourceStore.update() does a truncate-then-write with no tmp-file+rename (sources.py:61-67). During a distill loop the watcher will regularly wake mid-write — parse throws, or worse, a truncated file emits phantom "source removed" events.

accepted fix Two-sided: watcher treats parse-failure/shrunk-file as retry-after-debounce, AND the pipeline gains a 3-line atomic tmp-write+rename — an amended promise, not a silently broken one.
catch · material

Docs sources are born SCRAPED

Docs sources never pass FOUND (docs_pull.py:48) and skip DISTILLED entirely, jumping to INGESTED in bulk. In the polars workspace, 71 of 88 content-bearing sources have no distilled record — the designed found→scraped→distilled→ingested funnel would show a wall of "stuck at scraped" for the run's dominant category.

accepted fix Per-category lanes: docs lane = pulled → pushed; web lane = the full four stages; funnel rendered as counts-by-state per lane.
catch · threatening

263 vs ~5 citations

Measured on real artifacts: the deep-research report has 263 12-hex id citations (dossier works beautifully there) — but the polars and pydantic tool-graph reports have ~5–6 each, citing [synthesis/overview.md]-style pointers instead, a form no part of the design parsed. The core read-loop would render blank chrome on exactly the long-lived reference workspaces that matter most.

accepted fix Synthesis-pointer citations parsed as first-class (chip → capability index); the tool-graph skill amended now with the one-line id-citation contract.
catch · threatening

Hash-only manifest breaks the join

ingested.json keys pushed artifacts by content digest only — the atlas's provenance breadcrumb (graph node → source file) assumed a join that may not exist, and add-only re-pushes leave orphan TextDocuments from superseded revisions in the graph.

accepted fix An ordered plan gated by a 2-hour spike: check source_content_hash against manifest digests; else re-hash stable keys; orphans labeled "superseded revision"; pipeline urged to record Cognee data-ids at push time.

05

Critique & revision — what the adversarial pass changed

every finding got a verdict in 05-critique-response.md · §6 Revisions added to each design, and §6 wins

Two critics, two lenses: a skeptical product designer on flows and coherence, and a staff-engineer feasibility skeptic reading the designs against the actual code on 07-07-tool-graph and four real workspaces. Verdicts: overwhelmingly ACCEPT, four strategic MODIFY reframes, zero flat rejections. The highest-value changes, per surface:

drc mission

12 findings · 11 accept · 1 modify
Triage verbs live only in the UI — dismissed flags reappear on reload; the flagship loop loses its work. Directives file pulled forward to v1 as viz/directives.jsonl; every verb appends a directive record; queues derive from sources ⊖ directives.
Esc both exits the inspector and destructively marks-all-seen — one keypress wipes the delta you came back to review. Shared keymap: Esc = dismiss topmost pane, everywhere, always; mark-all-seen = explicit badge-chip click only.
One monotone funnel + F/S/D/I grid — a wall of "stuck at scraped" on tool-graph workspaces; "ingested" status can disagree with the manifest. Per-category lanes (docs: pulled → pushed; web: full four stages); grid's last column = pushed, derived from ingested.json, with disagreement as a visible cell state.
Watcher watches its own writes; events.jsonl grows forever; v1 in 2–3 days. viz/ excluded from the watch filter; 10MB rotation; v1 re-budgeted to 5–6 days. Pitch reframed: the durable core is grid + queues + delta — things a chat transcript is bad at.

drc atlas

13 findings · 12 accept · 1 modify
No empty state before cognify; the D5 co-occurrence graph mentioned nowhere in the surface. D5 becomes the atlas's pre-cognify layer and empty state: same scene UI over distilled/*.json, with a "local preview — cognify for the full graph" banner.
FA2 layout described as both server- and client-side; scene URLs break on re-cognify; diff mode reintroduces the global hairball. FA2 runs client-side in a web worker, positions persisted into the snapshot; scenes serialize declaratively (search + expansions + lenses) and re-resolve on load; diff is scene-scoped, entered via the "most rewired entities" list.
Provenance breadcrumb built on a hash-only manifest join that may not exist; orphan TextDocuments pollute counts. 2-hour spike decides the join design; orphans labeled "superseded revision" and excluded by default; pipeline urged to record Cognee data-ids at push time.
Graph browsing pitched as an equal peer despite being the weakest-demand job. Conceded in the pitch: atlas ranked 3rd of 3 for build order; provenance breadcrumb + gap cards named as its survival criteria; the spike gates the investment. v1 re-budgeted +5 days.

drc dossier

11 findings · 10 accept · 1 modify
Design assumes id citations; real tool-graph reports cite [synthesis/overview.md] pointers (~5 ids vs 263) — blank chrome in tool mode. Synthesis-pointer citations parsed as first-class (chip → capability index); tool-graph skill amended now; a 1-hour citation-coverage script runs before build.
VENDOR valence inferred by an uncomputable subject-entity heuristic; Admiralty-style grades added complexity. Valence ladder: parse the report's own labels → computable proxies (category, docs-domain) → no badge, never a guessed one. Grades cut entirely; weakest-first audit sort carries the signal.
Fuzzy-match viability asserted, not tested; docs sources (no DistilledRecord) had no card at all. rapidfuzz spot-test gates v0; distill quote-spans upgraded to a prerequisite for the exact-highlight tier; a dedicated docs-source card defined (title, URL, found_via, excerpt), excluded from valence.
"Reports are read once" — the retention story was weak. Reframed: read/audit is per-run value; retention lives in the capability index, whose Controls tab is promoted from v1.5 to v1 for tool workspaces. Tool mode re-budgeted +2 → +4 days.

Cross-cutting + gates

5 findings · 4 accept · 1 modify
Two surfaces named "Atlas"; Esc semantics diverge per surface; two provenance implementations can disagree. Dossier's tab renamed Controls; one keymap doc in the chassis spec; one shared provenance.py service with a single confidence vocabulary: exact / fuzzy(ratio) / revision-unknown / unresolved.
Three v1s bust the weekend budget; viz/ is unmanaged shared state. Shared-chassis v0 week adopted (see §7); every viz/ file gets a version field + retention rules; a single watcher owns invalidation.

The pre-build spike list · ~1 day total, gates ~3 weeks of builds

  1. Watcher torture test (½ day) — awatch vs a scripted SourceStore.update loop on a real workspace; count parse failures and phantom events. gates: mission
  2. source_content_hash join check (2h) — compare graph-export hashes against ingested.json digests on existing datasets. decides: atlas provenance design
  3. Citation-coverage script (1h) — id-regex + normalized-URL + synthesis-pointer extraction over the four existing report.md files. gates: dossier scope split
  4. rapidfuzz hit-rate (1h) — takeaway→content match ratios on agent-graph-knowledge-base. sets: dossier highlight-tier expectations

The shared-chassis v0 week

Instead of three sequential v1s (which bust the weekend budget), build one week of connected v0s on a single drc dash FastAPI chassis: watcher + events + grid + inspector (mission v0), the D5 scene (atlas v0), citation chips + hover cards (dossier v0) — ≈ 1 week total. Then real usage, not speculation, decides which v1 gets built next. The ~1-day spike list runs first and gates all of it.
