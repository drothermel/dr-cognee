act 2 · the moment

Three postures. One honest data problem.

A drc run ends with report.md, 34 SourceRecords, distilled takeaways and claims_of_use, and every scraped page on disk — a pre-built version of what NotebookLM, Elicit, and gwern.net compute at runtime. The only missing piece is presentational. But the reader arrives in three different postures, and the citations underneath them are not what the first design assumed.

Read

linear · after a run · default mode

Is this report right? What's the recommendation? Verification must be approximately free — margin notes carry the strongest evidence beside each claim, so the common case costs a glance.

"Can I trust this sentence?" — claim → evidence in ≤2 interactions.

Audit

skeptical · spot-checking · press a

Which claims are thin? The claim table inverts the report: rows are claims, sorted weakest first — uncited sentences and vendor-only rankings float to the top with warning tint.

"41 claims · 31 grounded · 7 thin · 3 uncited" — stated, not buried.

Reference

months later · tool-graph workspaces · Controls tab

"What were pydantic's serialization controls again?" The synthesis becomes a living capability index — every knob filterable, every entry keeping its doc pointer. This is the surface's retention claim.

Promoted from v1.5 to v1 by the critique — it earns the keep.

The first design assumed one citation grammar. Measurement said no — and the critique called it threatening. So the design now handles both, natively.

How the design absorbs it

  • 01Synthesis-pointer citations are first-class. A [synthesis/configuration-and-controls.md] pointer becomes a chip that opens the matching Controls-index section — not a dead string.
  • 02The tool-graph skill is amended now with the one-line id-citation contract, so future tool reports carry ids too.
  • 03A 1-hour citation-coverage script runs before any build — id-regex + normalized-URL + synthesis-pointer extraction over the four existing report.md files gates the scope split.
why this honesty is the pitch

A verification surface that fudged its own evidence base would be a joke at its own expense. This design was critiqued, measured, and re-budgeted: the VENDOR heuristic that couldn't be computed was cut and replaced with a ladder; Admiralty grades that added complexity without signal were cut entirely; tool mode's hidden cost was re-priced from +2 to +4 days as a second evidence model. The dossier practices the epistemics it renders.

act 3 · how it works

Two ladders and a tamed gap marker.

Everything in the dossier is pay-as-you-go depth. The detail ladder controls how much evidence you see; the valence ladder controls how much the UI dares to claim about that evidence; gap markers fire only where absence of citation actually means something.

act 6 · design decisions

Eight calls, each with its receipts.

Every decision below traces to the research (R5), the case study (CS3), or the critique response (05). Two of them are cuts — the pitch shows its dead code.

Margin notes visible by default

Gwern's finding is blunt: implementations requiring clicks "defeat much of the point." Sidenotes reclaim the margin a 70ch line length creates on a wide monitor. JS overlap reflow, density cap of one note per paragraph, overflow collapses into the chip count.

[gwern: visible by default][R5 §9: sidenotes on wide screens]

Panels, never modals

Report scroll position is sacred. NotebookLM's source viewer opens beside the answer; Perplexity's sources are a parallel region. The tri-panel keeps claim, distilled record, and source simultaneously visible — a wide monitor means we can be greedier than any of the exemplars.

[R5: panels beat modals][CS3: side-by-side persistence]

Admiralty grades — cut

The two-axis B2-style grade was researched, designed, and then deleted. Why: graders conflate the axes in practice (the known Admiralty critique), and a reliability letter derived from a category field is a guessed badge wearing a uniform. Valence + source count + the weakest-first sort carry the same trust signal with zero pseudo-precision.

[05: grades cut entirely][R5: axis-conflation warning]

The valence ladder, not a heuristic

The original VENDOR heuristic — infer the subject entity's domain — was uncomputable on real data, and both critics caught it. The replacement: (1) parse the report's own [vendor]/[practitioner] labels; (2) computable proxies only; (3) unlabeled → no badge, never a guessed one. Badge tooltips show derivation inputs — the scite lesson.

[05: valence ladder][scite: inspectable, never oracular]

The docs-source card

Docs sources skip distillation — they have no DistilledRecord to hover. Rather than render a broken card or mislabel them SECONDHAND, they get their own card: title, URL, found_via, first-heading excerpt from the content file — and they're excluded from valence entirely. A docs chip is a fact, not a judgment.

[05: docs card defined][gwern: annotation object per link]

Controls index promoted to v1

The skeptic's best shot — "a report is read once" — relocated the surface's retention claim. The capability index (renamed from "Atlas" to avoid the tab collision) moves from v1.5 to v1 for tool workspaces: months later, "what were pydantic's serialization controls?" is answered by a filterable index, not a re-run.

[05: read-once reframe][05: tab renamed Controls]

Gap markers only where they mean something

The first claim extractor would have flagged hedges and connective prose — crying wolf kills a warning channel. v1 fires on exactly one pattern: ranking-section sentences with zero citations. Vocabulary expands only after observing real reports, and the "indexed sentences" stat keeps extraction drift honest.

[05: cries-wolf fix][Wikipedia: citation needed, inline]

No dead canvas in Read mode

Before any chip is clicked, the evidence column would sit empty — dead pixels on the widest part of the screen. Instead, its idle state shows the strongest evidence for the current viewport, updating as you scroll: the panel teaches its own use before the first interaction.

[05: dead-canvas fix][CS3: keep all three panels open]

the machinery

ReportIndex · dr_cognee/dash/report_index.py

  • Pydantic model built on page load from report.md + sources.jsonl + distilled/*.json + content/*
  • Members: claims (sentence, cited ids, valence roll-up), citation→evidence map (takeaway + match span + confidence), gap list
  • Server-rendered markdown with a custom pass: id tokens → chips, claim index, margin-note data attributes
  • Client JS at Alpine scale, no build step: sidenote reflow (~80 lines), panel management, table sort

pipeline asks — two one-liners

Small edits, big downstream

  • Distill emits quote per takeaway/claim — upgraded from recommendation to prerequisite for the exact-highlight tier
  • Report skill keeps the [src_xxxx] citation form — one sentence in the skill, much less tolerant-parsing risk
  • Tool-graph skill amended now with the id-citation contract (accepted in the critique response)
  • Later, optional: per-claim valence from the distill LLM — heuristic kept as fallback forever
risks, priced

What can go wrong, and what catches it

  • Fuzzy match misleads → always show confidence; below threshold, no highlight rather than a wrong one
  • Valence misfires → tooltips show derivation inputs; errors surface where scrutiny already is (the weakest-first sort)
  • Report format drift → best-effort per sentence; the "indexed sentences 38/41" stat keeps degradation visible
  • Shared chassis → one provenance.py, one confidence vocabulary: exact / fuzzy(ratio) / revision-unknown / unresolved; Esc = dismiss topmost pane, everywhere
act 8 · what the critics said

The two best objections — and what they changed.

Both were accepted or absorbed. A pitch that can't show its strongest counterargument isn't a pitch for a verification tool.

the skeptic · strongest objection
"A report is read once. You're building a cathedral for a document with a 20-minute lifespan."
response · verdict: modify — pitch reframed, design changed

The per-run value is real and stands on its own: that 20-minute read is exactly where a wrong ranking gets adopted or caught, and this surface makes catching it cost 2 hovers and a click instead of thirty tab-switches. The 263-citation report is unreadable-with-verification today; it's the primary artifact of the whole pipeline.

But the retention claim moved. It now lives in the Controls capability index — the piece you return to months later — which was promoted from v1.5 to v1 for tool workspaces precisely because of this objection. Read/audit is the per-run pitch; Controls is the shelf life.

both critics · material finding
"Your VENDOR heuristic — match the source URL against the subject entity's domain — is uncomputable. Nothing in the data model tells you which entity a claim is about."
response · verdict: accept — heuristic deleted

Correct, and it shipped a better design. The subject-entity inference is dropped entirely. In its place, the valence ladder: (1) the report's own [vendor]/[practitioner] labels — which the agent already writes — parsed as ground truth; (2) computable proxies only (source category, docs-domain); (3) unlabeled means no badge.

The same review also cut the Admiralty grades (complexity without signal) and re-priced tool mode at +4 days. Every one of those decisions is encoded in design §6, which supersedes the original sections where they conflict.
