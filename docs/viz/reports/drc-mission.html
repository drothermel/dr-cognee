drc mission
the Run Cockpit

watch the run · trust the agent · steer between iterations

A drc run is an AI agent marching 20–300 sources through harvest → scrape → distill → synthesis → ingest for tens of minutes, while you do other work. The cockpit is one live, local screen — plain files in, SSE out — that answers the re-engagement question in a glance: "I've been away. What happened, is anything stuck, where should I steer next?"

It borrows exactly three production-proven organs — Airflow's dense status matrix, Temporal's "is it wedged?" Now panel, and GitHub Actions' hoisted-error triage funnel [CS1] — and mounts them over data the pipeline already writes. No database. No build step. One process.

act 02 · the moment

One human, one agent, tens of minutes of unattended machinery

During a run the agent operates drc; the human tabs in every few minutes. The cockpit optimizes for that re-engagement moment — not for operating controls, not for replacing logs, not for post-hoc analytics.

Who's watching

One power developer on a ~3800px-wide monitor, running deep-research and tool-graph flows through an agent. She is the tool's owner, its only user, and its maintainer — so zero-maintenance beats polish: the design refuses any component that needs care between runs [R7: files+SSE stack].

Her attention pattern is bursty: 10 seconds of glancing per 5 minutes of absence, with one deliberate triage sit-down between iterations. Every design choice below is priced against that budget.

Non-goals, stated up front: not a control panel (the agent drives drc), not a log viewer (raw logs are an escape hatch, not the surface), not retrospective analytics (M7 is honestly cut — the event log accrues the data for a future replay tool).

The five questions, ranked

  1. Is the agent working right now, or wedged?trust · Now panel
  2. What broke while I was away?errors · hoisted strip
  3. What changed since I last looked?delta · violet badges
  4. Where is coverage thin / which depth flags matter?steering · queues
  5. What is the run learning?pulse · entities

v0 nails Q1–Q2 in a day. v1 answers all five. Green outlines mark the two questions nothing else in the toolchain can answer at all.

act 03 · how it works

Three loops on top, one event log underneath

The UI is three attention loops sharing one screen; the plumbing is one watcher minting semantic events from files the pipeline already writes. Nothing else.

act 06 · design decisions

Eight decisions, each with receipts

Every load-bearing choice traces to the research base: R1 (live observability), R6 (corpus triage UX), R7 (local web-app stacks), CS1 (oversight-cockpit anatomy), S2/S3 (synthesis), and the critique round.

D·01Counts-by-state. Never percent bars.
Harvest keeps growing the denominator; a percent bar would oscillate and lie. Counts per state, per lane, each one a click-filter — progress reads as flow through the funnel, not fraction of an unknowable total.
[R1: pattern 8][§2 P3]
D·02The grid never leaves the screen.
Airflow's core move: detail rotates through a sidebar while the matrix holds context. Click a cell → inspector fills; zero navigation, zero context loss. Hover→click→drill is the entire interaction grammar.
[R1: Airflow grid][CS1]
D·03Errors are hoisted, and the failing stage auto-expands.
Airflow's known weakness is that why lives two clicks deep. GitHub Actions' fix — annotations up top, failed step pre-opened — makes required interactions to see error text zero-to-one. The cockpit steals it whole.
[CS1: Actions annotations]
D·04The Now panel answers "wedged?" — and reports observables, not diagnoses.
Temporal's Pending Activities panel kills the most anxiety-producing question by making retry state first-class. The stall warning says "no file activity for 190s" — never "stuck" — with the threshold scaled per event kind, because a thinking agent isn't a wedged one.
[CS1: Temporal][§6: per-kind threshold]
D·05Per-category lanes, and the last column is manifest truth.
Docs sources are born scraped and skip distill — one universal funnel would render tool-graph runs as permanently broken. Docs lane = pulled→pushed; web lane = the full four. And pushed derives from ingested.json keys, with status-vs-manifest disagreement as a visible state.
[§6: lanes][critique: ACCEPT ×2]
D·06Zero-target queues with persisted verbs.
Screening tools converge on the same triage shape: a queue you drive to zero, keyboard verbs that act-and-advance, undo within reach. Every verb appends to viz/directives.jsonl — queues derive from sources ⊖ directives, so a reload never resurrects finished work.
[R6: zero-target queues][§6: triage persists]
D·07Files in, SSE out, nothing in between.
The workspace is already watchable — everything but Cognee is plain files in one directory. FastAPI + watchfiles + htmx-over-SSE means no DB, no build step, no daemon beyond drc dash. The watcher mints events.jsonl because sources.jsonl is a rewritten state doc, not a stream: full reparse, id→hash diff, semantic transitions.
[R7: files+SSE stack][S2][§2 P1–P2]
D·08Entity pulse: the graph preview before the graph exists.
Entities across distilled records form a co-occurrence network before Cognee runs — with perfect provenance back to source ids. As a ranked-bar panel it answers Q5 ("what is it learning?") for free, and it's the same D5 layer the atlas uses as its pre-cognify empty state. One data derivation, two surfaces.
[S3: D5 underexploited][00-framework: D5]

act 08 · what the critics said

The design was attacked before it was pitched

Two critique agents (UX + feasibility) plus a skeptic pass ran against the draft. Every finding got a verdict — ACCEPT, MODIFY, or REJECT with reasons — and the accepted ones are already folded into everything above. Showing the scars is part of the pitch.

the strongest objection · skeptic
"The agent chat next door already does this. You can just ask it what happened. Why build a dashboard for a conversation you're already having?"
verdict: modify — reframe, not redesign

Conceded where it's true: for "what are you doing right now?", the chat is genuinely competitive, and the Now panel and feeds are honestly labeled as cheap by-products of a watcher that exists anyway. But the objection dies on three counter-examples: a transcript cannot show the shape of 134 sources at once, cannot diff itself against your last visit, and cannot hold a persistent triage worklist with undo. Asking the agent to reconstruct any of these burns tokens and minutes to produce, every time, a worse version of a screen that renders in milliseconds — and its answer is a claim, where the grid is evidence read directly from the files. So the value claim narrowed to exactly what survives: grid + queues + since-you-left delta. v0 builds only that. [05-critique: skeptic → MODIFY]

THREATENING

Torn reads of sources.jsonl

The pipeline rewrites sources.jsonl in place; the watcher can catch a half-written file, parse garbage, and emit phantom source-removals that make the grid flicker with lies.

FIX — ACCEPTED, TWO-SIDEDWatcher treats parse-failure or a shrunk file as retry-after-debounce and never emits removals from a failed parse; AND SourceStore.update gains the atomic tmp-write + rename. Belt, suspenders, and a torture-test spike to prove both.
THREATENING

Triage verbs didn't persist

In the draft, dismissals and chases lived in browser memory; a reload resurrected every flag you'd already handled. Ten minutes of triage work, evaporated — the kind of betrayal that kills a personal tool.

FIX — ACCEPTEDviz/directives.jsonl pulled forward from v1.5 to v1. Every verb appends a record (dismiss/snooze are directives to the dashboard, same schema); queues derive from sources ⊖ directives.
MATERIAL

One funnel shape was wrong for half the runs

Tool-graph workspaces are dominated by docs sources, which are born scraped and never distill — the draft's single found→scraped→distilled→ingested funnel would show them permanently stalled at two stages.

FIX — ACCEPTEDPer-category lanes (docs: pulled→pushed; web: all four), verified against docs_pull.py and ingest.py. Bonus fix from the same review: "ingested" status can lie, so the pushed column reads ingested.json instead.
ALSO CAUGHT · MATERIALWatcher watching its own viz/ writes (excluded); events.jsonl unbounded (rotate 10MB×2); Esc doing two jobs (one keymap: dismiss only); stall threshold crying wolf on slow distills (per-kind scaling); "v1 in 2–3 days" (re-budgeted to 5–6).
WHAT THE COCKPIT
STILL CAN'T DO

History starts when the watcher starts — "observed since 14:02" is printed on the chrome, not hidden. If the cockpit wasn't running, deltas and timeline degrade while the grid stays correct (it's state-based). Stall detection reports file-activity gaps, not agent cognition. And the steering loop is only as good as the skill honoring directives.jsonl — until v1.5 lands that sentence, the copy-paste steering note is the honest channel. None of these are footnotes; they're the shape of the tool.
